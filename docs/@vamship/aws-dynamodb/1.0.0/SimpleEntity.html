SimpleEntity

SimpleEntity

Extension of the Entity class with implementations for common CRUD operations. This is an opinionated implementation that injects audit tracking fields and a field to support logical deletes. Methods are provided for physical deletes if necessary.

The entities are by design lightweight, and do not perform too many validations, like checking for data types on specific fields, user authorization, etc. It is assumed that the caller of this module (like a Lambda function) will perform these tasks.

This class is intended to serve as a base class for specialized entity classes that will implement multiple properties on the base class, including, but not limited to tableName, hashKey, rangeKey (optional), updateProps, and updateProps.

Returns a list of entities that match the hash key.
Parameters:
Name Type Attributes Default Description
keys EntityKeys A set of key(s) that will be used in the list query. All queries will use the hash key to fetch a list of records. If a range key is specified, it will be used to generate a continuation token for the list query. The continuation token, taken in conjunction with the count option can be used to perform paged fetches.

If omitted, all records will be returned starting from the first record in the table.

count Number <optional>
 The number of records to return in a single fetch operation. If omitted, all records for the entity will be returned.

 {} Audit information to associate with the query.
