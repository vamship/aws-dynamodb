EntityKeys

Source:
  • entity.js, line 59
A set of keys that uniquely identify the entity record in the table. This object can define two properties:
  • hashKeyName: This property must have the same name as the value of hashKeyName
  • rangeKeyName: This property must have the same name as the value of rangeKeyName. The range key name property can be undefined, depending on the type of query to be performed, and/or the configuration of the entity object.

The values of these properties can be either numbers or strings.

Type:
  • Object
Example
// Assume that the entity has a hash key named "accountId" of type
// string, and a range key named "entityId" of type number.

const keys = {
 accountId: 'myAccount',
 entityId: 1001
};
